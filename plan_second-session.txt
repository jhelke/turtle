## 1:30-2:30 - Run controlled parallel tunnels

Now run two turtles in parallel, but with spacing.

Do not start them directly next to each other. Give them room.

Recommended pattern:

```text
Main corridor
     |
     +-- tunnel A: miner-01, tunnel 150
     |
  8-12 blocks spacing
     |
     +-- tunnel B: miner-02, tunnel 150
```

Example:

```text
Top view:

Main corridor
|
+---- miner-01 tunnel 150
|
|  10 blocks
|
+---- miner-02 tunnel 150
```

Why 8-12 blocks?

```text
- avoids overlapping tunnels
- keeps exploration clean
- easier to mark
- easier to recover turtles
- enough separation to expose useful ore/caves
```

Operational loop:

```text
[ ] Put coal into turtle
[ ] Check fuel
[ ] Start tunnel 150
[ ] While it runs, process/sort previous output
[ ] When it returns, unload
[ ] Refuel
[ ] Repeat or move to next branch
```

Do not over-optimize yet. Your real gain comes from running **two boring reliable turtles**.

## 2:30-3:00 - Make your first tiny helper script

Since you have not custom-scripted yet, start with something harmless: a status script.

Create on each turtle:

```lua
edit status
```

Paste:

```lua
print("Label: " .. os.getComputerLabel())
print("ID: " .. os.getComputerID())
print("Fuel: " .. turtle.getFuelLevel() .. " / " .. turtle.getFuelLimit())

local used = 0

for slot = 1, 16 do
  local count = turtle.getItemCount(slot)
  if count > 0 then
    used = used + 1
    local detail = turtle.getItemDetail(slot)
    local name = detail and detail.name or "unknown"
    print("Slot " .. slot .. ": " .. count .. "x " .. name)
  end
end

print("Used slots: " .. used .. " / 16")
```

Run:

```lua
status
```

This is a good first script because:

```text
- it cannot lose the turtle
- it cannot move the turtle
- it teaches reading turtle state
- it helps before and after every mining run
```

Optional second helper:

```lua
edit fuelall
```

```lua
for slot = 1, 16 do
  turtle.select(slot)
  turtle.refuel()
end

turtle.select(1)
print("Fuel: " .. turtle.getFuelLevel() .. " / " .. turtle.getFuelLimit())
```

Use it only when you intentionally put fuel in the turtle inventory.

## 3:00-3:30 - Standardize the mining protocol

Write this on an in-game sign or keep it next to you:

```text
Turtle mining protocol:

1. Place turtle at marked dock.
2. Confirm direction.
3. Add coal.
4. Run status.
5. Run tunnel 150.
6. Wait for return.
7. Empty inventory.
8. Refuel if needed.
9. Mark tunnel complete.
```

For each finished tunnel, mark it:

```text
Tunnel A1 - complete
Tunnel A2 - complete
Tunnel B1 - complete
```

Good branch naming:

```text
A1, A2, A3 = north-side branches
B1, B2, B3 = east-side branches
C1, C2, C3 = future lower level
```

This sounds silly, but once you have 3 turtles running, names prevent confusion.

# Avoid this in session 2

```text
Do not run three turtles blindly in different directions.
Do not start GPS yet.
Do not start rednet command/control yet.
Do not write a custom excavator yet.
Do not mine so far that recovery becomes annoying.
Do not let fuel/output handling stay informal.
```

The big win now is not clever scripting. It is **operational discipline**.

You want this loop to become boring:

```text
dock -> fuel -> status -> tunnel -> unload -> sort -> repeat
```

Once that loop is reliable with 2-3 turtles, custom scripts become much easier and much safer.
